Install


Hello! This section will show you how to install your login system. First of all, you should keep this page up, and then (if this system isn't already uploaded) upload this system onto your website. Then you should go to the install.php file on your site in a new window/tab. While you read the Install section, the actual system should not be installed, as it acts as a walk-through.

Database


Setting up your database is quite easy. Fill in the necessary fields, the port field does not need to be filled in unless you know you must, if you have no idea what it is for, don't try to fill it in. When the installation is complete, you will be asked to download a file called config.php please download it and upload it into the /includes directory. If you ever need to change your database settings, they will be stored in there. Remember keep the install.php file opened in the other browser while you read this section.

Files/Directories


It will be necessary for you to chmod certain files and directories so they will work properly. The /templates directory should be chmoded to 666 and the /compiled directory should be chmoded to 771. Now you can go into the /language/en folder and chmod the file en_register_email.txt and en_profile_email.txt to 755. Remember keep the install.php file opened in the other browser while you read this section.

Other Options


First should be the cookie settings. These aren't just for cookies, they are for sessions also so don't ignore them, or your login system won't work.
  • The cookie prefix field will allow all the cookies/sessions set by your site to have a prefix, so they don't clash with other cookies/sessions.
  • The cookie secure field will allow the cookies you set to be secure, this will only work if you use SSL (e.g. your site starts with https:// instead of http://).
  • The cookie path field sets the path to which the cookie can be read from, if you want your entire site to be able to read them, put / if you want just the directory that holds the login pages, put something like this /login_directory.
  • The cookie expiration is how long cookies will last before they expire (in seconds), which means the user could be logged out. If you want them always to be logged in, just put 31449600 (one year) in the field.
  • The cookie domain is the domain to which the cookie is being set, no http:// or www is needed. If you want subdomains to be able to read your cookies, you must put a dot ( . ) before it (e.g. .example.com).

Next should be some login settings.
  • The require activation determines whether or not you want the users to activate their accounts before logging in.
  • The max login attempts is how many times a user can login before not being able to for 12 hours (this is for security for trying to stop people to crack accounts). The default value is 5.
  • The login redirect is the page location of which the user will be redirected on a successful login, it can be a URL (e.g. http://www.example.com/page.php) or relative to the current directory (e.g. index.php).
  • The logout redirect is the page location of which the user will be redirected on a successful logout. It can be a URL (e.g. http://www.example.com/page.php) or relative to the current directory (e.g. index.php).

Now there are just some misc. options, which are each explained below.
  • The template directory is the directory of which the templates are stored, the default is templates, and it only needs to be changed if that directory is renamed.
  • The compiled directory is the directory of which the compiled templates are stored, the default is compiled, and it only needs to be changed if that directory is renamed.
  • The auth guest profile field is to determine whether guests can view member profiles or not.
  • The default template field is the template directory of the current template installed, it should be default as that is the default template given by me. But it can be changed later when you install more.
  • The max about you field is the maximum length a user's about you section of their profile can be. This should be no higher than 20000.
  • The auth profile image field is the field to determine whether your users can place an offsite image onto their profile.
  • The password complexity (numbers) field lets you decide whether or not users must have at least 1 number in their password.
  • The password complexity (special characters) field lets you decide whether or not users must have at least 1 special character in their password.
  • The language field lets you decide what language the system will use, it can be changed but most likely won't.
  • The site name field is so you can put in a name for your site, that will be included in emails sent by the site so people who registered/recieve mail know who it is from.
  • The email signature field is so you can put a signature at the end of each email sent out by the site.
  • The from email is the email address you want users to see when they recieve email from you.
  • The gzip field is to see whether or not you want GZIP Compression enabled for your site. GZIP Compression can speed up the output of HTML by your site as it compresses the page. The option will not show if the zlib library is not found on your server - sorry.

Configuration


This section will show you how to configure these files so they work the way you want them to. First I will show you how to set up members only pages.

Members Only


So you want to know how to setup a members only page eh? This section is for you then. First of all, you will need to open the following files:
  • /layout.php
  • /includes/header.php

Okay, open the first file in a text editor. Make sure you use an editor that supports LF endings. You can tell if yours doesn't as boxes will appear all over the place. If you use Windows, I can suggest Notepad2, it is what I use to code all my products. Anyway moving on once you are in this file, you will notice it is mostly blank. You should see a piece of code like this:

/* Look in the header.php file for the next page number so you can define the title */
define('PAGE', 7);
include_once('includes/header.php');

include_once('includes/footer.php');

You should notice there is a space between the first and second include. This is where our code will go. Put this code in to determine whether the user is logged in or not:

if ($is_logged) {
?>

<?php
exit;
}
else {
$tmpl->display('not_logged.qtmpl');
}

Now in this same file (with new content added) right below define('IN_LOGIN_SYS', true); you should put this code:

define('HEADER', false);
define('GUEST', true);

This will stop the header made by me to be output if the user is logged in, allowing you to make the HTML custom to your own site. It will also make sure the header is output if the user is a guest, to keep the login system intact. If you want to build onto the current layout, you must remove the exit; command, and then change define('HEADER', false); to define('HEADER', true);. Now we can define the page title (if you are using the already made layout). You should see the line define('PAGE', 7);, this is where the admin.php file comes into play. In the admin.php file down bottom, you will see a list of page numbers with there titles, all you have to do is add onto it, like so:

case 8:
$page_title = 'My Custom Member Page';
break;

Now save the admin.php file and upload it. Now you can add your HTML to this file, that will go between ?> and <?php. Now you can save this file as mynewpage.php in the same directory as the other files. After uploading it, it should work fine.
